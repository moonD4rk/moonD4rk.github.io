看了列表，可能会有疑问：为什么 MacOS 平台导出浏览器数据码下需要密码，而 Windows 不需要呢？为什么 Chrome for WIndows 要分为 80 版本以下和以上呢？

密码文件存储位置

Chrome 的数据文件存储路径可以通过在搜索框中输入 chrome://version 看到，其中 Profile Path 就是存储路径。

此路径下有 Cookies History Bookmarks Login data 这四个文件用来存储对应的信息，拷贝后使用 sqlite3 数据库打开Cookies ，看到 encrypte_value 关键字段是 BLOB (Binary long Object) 类型，我们要做的就是如何解密这串二进制。

Chromium 密码加密方式

Chromium是Google为发展浏览器Google Chrome而开启的免费开源软件项目，以BSD许可协议等数种许可发行并开放源代码。Chromium与Google Chrome共享大部分代码和功能，但功能和商标之间有一些细微差别。

由于 Chrome 和 Chromium 共享大部分代码，而 Chromium 的代码是开源的，各个平台具体的加密所用代码可以在官网直接看到。

Chrome for Windows

Chrome for WIndows 在 80 版本之前调用 win32 接口 DPAPI（data protection API） 来加解密。

在 80 版本及以后使用的是 AES-256-GCM 算法，而 AES-256-GCM 需要用到 secret key 值存储在 Profile Path 上级路径的 Local State 文件中，字段 encrypted_key 为 AES 用到的 key 值。整个解密过程无需任何密码，用户只需可以访问 Chrome 数据文件便可完成解密。

Chrome for MacOS

Chrome for MacOS 上的解密有些稍微不一样，通过源码可知用的是 AES-128-CBC算法，其中用到的 secret key 值则存储在 Mac keychain中，我们在 Keychain Access.app 中搜索 chrome 可以看到 Chrome Safe Storage ，点击 Show password 得到的值为 AES 所需 key，整个过程必须得要输入系统密码。

访问Keychain Access 的内容苹果提供了命令行的方式，Terminal 输入 security find-generic-password -wa 'Chrome'，还是会弹出输入登录密码才允许访问 key。从获取 Key 的过程看，MacOS 安全性要比 Windows 高不少…

使用 Chromium 的其他浏览器

目前市面上绝大多数浏览器都是基于 Chromium 开发，大部分都可以通过以上方式来解密，但也有一些例外。经过简单测试， 360安全浏览器 和 搜狗浏览器 是自己实现了加密算法，而QQ browser for Mac 甚至没有存储密码的功能。

总结

粗略介绍了基于 Chromium 浏览器密码的加解密方式，后面会针对 Firefox 和Safari 的解密写一篇介绍文章。文中提到的 win32接口 、 AES-256-GCM 和 AES-128-CBC 及对应 secret key 的具体使用代码可查看本项目Github源码。
